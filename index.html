May 10, 2026.I've discovered Reservoir Computing thanks to a video by Artem Kirsanov and a blog post by Ben Hell about a bucket of water. I'm fascinated! The idea is to feed an input into a fixed, randomly initialized recurrent network (spiking or ANN) and only train the weights of a linear output layer. If it's setup correctly, the interactions of the random network nodes will create "higher order" features based on the input signal. Then the output layer's optimization function can decide which of these features are relevant to the target task (classificaion, regression, etc). But it doesn't have to be a digital network. A reservoir computers can be made from analog circuits, a bucket of water, or even a biological neural network such as a brain organoid. According to Jaeger, who came up with the Echo State Network variant, the network works best when it runs on "the edge of chaos". If it is this controlled chaos that makes it work, it raises the question: is human creativity also a result of our brains running at the edge of chaos?

April 14, 2026. I've discovered ShinkaEvolve via an episode of Machine Learning Street Talks. The idea is to evolve a population of programs to optimize a metric.

John Koza explored this idea in the 90s with what he called Genetic Programming. The idea is to have a population of s-expressions (of course it was LISP back then), evaluate each s-expression against a value function, and replace some portion of the worst performing programs with mutated versions of the best performers. Over time, the population as a whole wiggles towards better performance.

ShinkaEvolve leverages LLMs by giving them the task of intelligently mutating the programs. This means you aren't restricted to an s-expression tree; you can use any language the LLM understands and tie in any library you need. This opens quite a lot of options! ShinkaEvolve appears to use some sort of orchestration to efficiently shepherd the population towards solutions. I'm still exploring how it works.
